## Overview

The platform generates events at multiple layers. The analytics pipeline aggregates these into actionable metrics.

```
┌─────────────┐    ┌─────────────┐    ┌─────────────┐
│   Sources   │───▶│  Pipeline   │───▶│  Metrics    │
│  (Events)  │    │  (Rollups)  │    │  (Queries)  │
└─────────────┘    └─────────────┘    └─────────────┘
```

| Source | Events Generated |

| Source | Events Generated |

## Analytics Pipeline Design (Planned)

### Event Ingestion

```
1. Event emitted by service
2. Async write to domain_events table
3. Event published to queue (optional, for heavy processing)
```

### Rollup Triggers

| Rollup Type | Trigger | Frequency |
|-------------|---------|------------|
| Hourly counts | Cron | Every hour |
| Daily counts | Cron | Every day |
| Campaign summary | `campaign.completed` event | Immediate |
| Workspace usage | Message send | Immediate |

### Materialized Views (Planned)

```sql
-- hourly_message_counts (refreshed hourly)
CREATE MATERIALIZED VIEW hourly_message_counts AS
SELECT
    workspace_id,
    DATE_TRUNC('hour', created_at) as hour,
    status,
    COUNT(*) as count
FROM message_events
GROUP BY workspace_id, DATE_TRUNC('hour', created_at), status;

-- daily_workspace_usage (refreshed daily)
CREATE MATERIALIZED VIEW daily_workspace_usage AS
SELECT
    workspace_id,
    DATE_TRUNC('day', created_at) as day,
    SUM(CASE WHEN status = 'sent' THEN 1 ELSE 0 END) as messages_sent,
    SUM(CASE WHEN status = 'delivered' THEN 1 ELSE 0 END) as messages_delivered
FROM message_events
GROUP BY workspace_id, DATE_TRUNC('day', created_at);
```

## Projections

### Contact Engagement Score

**Source Events**: `message.sent`, `message.delivered`, `message.read`, `contact.created`, `contact.updated`

**Projection Logic**:
```python
def compute_engagement_score(contact_id: int, days: int = 30) -> float:
    sent = count_events(contact_id, 'message.sent', days)
    delivered = count_events(contact_id, 'message.delivered', days)
    read = count_events(contact_id, 'message.read', days)
    return (sent * 1 + delivered * 2 + read * 5) / days
```

**Refresh**: On `message.sent` event (real-time) or batch job (hourly)

### Campaign Funnel

**Source Events**: `message.sent`, `message.delivered`, `message.read`, `message.failed`

**Projection Table**:
```sql
campaign_funnels:
  - campaign_id
  - sent_count
  - delivered_count
  - read_count
  - failed_count
  - funnel_rate (delivered/sent)
  - engagement_rate (read/delivered)
  - last_updated
```

**Refresh**: On delivery status webhooks (real-time)

### Workspace Usage

**Source Events**: `message.sent`

**Projection Table**:
```sql
usage_tracking:
  - workspace_id
  - messages_sent
  - billing_cycle (YYYY-MM)
  - last_updated
```

**Refresh**: On every send (immediate, via `ensure_workspace_can_send`)

## Realtime vs Batch

### When to Use Realtime

| Scenario | Approach |
|----------|----------|
| Campaign progress bar | Poll `campaign_contacts` aggregates |
| Delivery status updates | Webhook → `message_tracking` |
| Quota enforcement | Immediate write to `usage_tracking` |

### When to Use Batch

| Scenario | Approach |
|----------|----------|
| Historical analytics | Cron rollup jobs |
| Contact engagement scoring | Hourly batch job |
| Weekly reports | Daily aggregate → email |
| Archival | Monthly cleanup job |

### Hybrid Approach

Example: Contact engagement score

```
1. Real-time: Increment counters on send/delivered/read
2. Batch: Nightly recompute scores from raw events
3. Query: Return cached score with "updated_at" timestamp
```

## Query Patterns

### Delivery Rate Query

```sql
SELECT
    campaign_id,
    COUNT(*) as total,
    COUNT(*) FILTER (WHERE status = 'sent') as sent,
    COUNT(*) FILTER (WHERE status = 'delivered') as delivered,
    ROUND(
        COUNT(*) FILTER (WHERE status = 'delivered') * 100.0 /
        NULLIF(COUNT(*) FILTER (WHERE status = 'sent'), 0),
        2
    ) as delivery_rate_pct
FROM message_events
WHERE workspace_id = $1 AND campaign_id = $2
GROUP BY campaign_id;
```

### Timeline Query

```sql
SELECT
    DATE(created_at) as date,
    COUNT(*) FILTER (WHERE status = 'sent') as sent,
    COUNT(*) FILTER (WHERE status = 'delivered') as delivered,
    COUNT(*) FILTER (WHERE status = 'read') as read,
    COUNT(*) FILTER (WHERE status = 'failed') as failed
FROM message_events
WHERE workspace_id = $1
    AND created_at >= NOW() - INTERVAL '30 days'
GROUP BY DATE(created_at)
ORDER BY date;
```

### Engagement Query

```sql
SELECT
    c.id, c.name, c.phone,
    COUNT(me.id) as total_events,
    COUNT(me.id) FILTER (WHERE me.status = 'sent') as sends,
    COUNT(me.id) FILTER (WHERE me.status = 'delivered') as deliveries,
    COUNT(me.id) FILTER (WHERE me.status = 'read') as reads,
    MAX(me.created_at) as last_interaction
FROM contacts c
LEFT JOIN message_events me ON me.contact_id = c.id
WHERE c.workspace_id = $1
GROUP BY c.id, c.name, c.phone
ORDER BY reads DESC, last_interaction DESC;
```
